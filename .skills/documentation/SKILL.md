# Documentation

When this skill is active:

- Write for a developer who is seeing this code for the first time.
- For functions/methods: describe purpose, each parameter (name, type, meaning), the return value, and any errors thrown.
- Prefer short paragraphs and bullet lists over long prose blocks.
- Include one minimal, runnable usage example when documenting a function, class, or CLI command.
- Do not invent behavior that isn't shown in the provided code — say so and ask for clarification instead of guessing.
